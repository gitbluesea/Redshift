## 보안 채널인 SFTP 대신 일반 FTP(기본 21번 포트)를 사용하여 원격 서버로 데이터를 전송하는 Shell 스크립트입니다.
FTP는 SSH Key 같은 인증 방식이 없기 때문에, 스크립트 자동화를 위해 아이디와 비밀번호를 스크립트 내에 명시하여 접속하는 방식을 사용합니다.
대용량 처리를 고려하여 Redshift에서 S3로 내보낼 때 GZIP 압축을 진행하고, 스크립트 실행 서버의 디스크로 내려받은 뒤 곧바로 FTP 전송 후 삭제하도록 구성했습니다.

### 📋 전체 Shell 스크립트
Linux 표준 FTP 클라이언트인 ftp 명령어를 활용한 스크립트입니다.
```
#!/bin/bash
# 에러 발생 시 즉시 스크립트 중단
set -e

# ==========================================
# 1. 환경 설정 (사용자 환경에 맞게 수정)
# ==========================================
# Redshift 설정
export PGPASSWORD="your_redshift_password"
REDSHIFT_HOST="://amazonaws.com"
REDSHIFT_PORT="5439"
DB_NAME="your_db"
USER_NAME="your_user"

# S3 및 AWS 설정 (Redshift가 접근할 IAM Role)
S3_BUCKET="s3://your-bucket-name/unload_data/"
IAM_ROLE="arn:aws:iam::123456789012:role/YourRedshiftRole"

# 로컬 임시 디렉토리 (스크립트 실행 서버 내부 디스크)
LOCAL_TEMP_DIR="/tmp/redshift_unload_$(date +%Y%m%d%H%M%S)"
mkdir -p "$LOCAL_TEMP_DIR"

# 원격 FTP 서버 설정
FTP_HOST="://remote-server.com"
FTP_PORT="21"                                 # 기본 포트 21, 다르면 수정
FTP_USER="ftp_username"
FTP_PASS="ftp_password"                       # FTP 비밀번호 명시
FTP_REMOTE_DIR="/path/to/remote/target/dir"   # 목적지 디렉토리

# ==========================================
# 2. Redshift UNLOAD 실행 (S3로 내보내기)
# ==========================================
echo "[1/4] Redshift에서 S3로 UNLOAD 시작..."

psql -h "$REDSHIFT_HOST" -p "$REDSHIFT_PORT" -d "$DB_NAME" -U "$USER_NAME" -c "
UNLOAD ('SELECT * FROM large_table WHERE created_at >= \'2026-01-01\'')
TO '$S3_BUCKET'
IAM_ROLE '$IAM_ROLE'
FORMAT AS CSV
GZIP
ALLOWOVERWRITE
MANIFEST;
"
echo "-> UNLOAD 완료."

# ==========================================
# 3. S3에서 로컬 임시 디렉토리로 다운로드
# ==========================================
echo "[2/4] S3에서 AP 서버 로컬 디렉토리로 다운로드 중..."
aws s3 cp "$S3_BUCKET" "$LOCAL_TEMP_DIR" --recursive
echo "-> 다운로드 완료. (위치: $LOCAL_TEMP_DIR)"

# ==========================================
# 4. FTP를 통해 원격 서버로 전송 (모드 및 대화형 설정 적용)
# ==========================================
echo "[3/4] 원격 서버($FTP_HOST)로 FTP 전송 시작..."

# 임시 디렉토리로 이동하여 FTP 전송 준비
cd "$LOCAL_TEMP_DIR"

ftp -n -v "$FTP_HOST" "$FTP_PORT" <<EOF
user "$FTP_USER" "$FTP_PASS"

# [해결책 1] mput * 실행 시 y/n 대화형 묻기 비활성화 (Interactive mode Off)
prompt off

# [해결책 2] Active 또는 Passive 모드 설정 (원하는 모드 '한 줄만' 주석 해제하여 사용)
# ------------------------------------------------------------------
# 방법 A: Passive(수동) 모드 사용 시 (대부분의 클라우드 및 방화벽 환경 권장)
passive on

# 방법 B: Active(능동) 모드 사용 시 (사내 망 등 특수 환경)
# passive off
# ------------------------------------------------------------------

# 바이너리 모드로 전환 (파일 깨짐 방지)
bi

# 원격지 디렉토리 이동 및 전송
cd "$FTP_REMOTE_DIR"
mput *
bye
EOF

echo "-> FTP 전송 완료."

# ==========================================
# 5. 로컬 임시 파일 및 S3 데이터 정리
# ==========================================
echo "[4/4] 임시 파일 정리 중..."

# AP 서버 디스크 공간 확보를 위해 다운로드했던 파일 제거
rm -rf "$LOCAL_TEMP_DIR"

# (옵션) S3 버킷 내의 임시 데이터도 지우고 싶다면 아래 주석을 해제하세요.
# aws s3 rm "$S3_BUCKET" --recursive

echo "🎉 모든 프로세스가 성공적으로 완료되었습니다."
```

### 💡 FTP 전송 시 중요 체크포인트

   1. ``bi`` (Binary 모드) 명령어
      * FTP 전송 중 텍스트 파일이나 압축 파일(GZIP)이 깨지는 것을 막기 위해 반드시 bi 또는 binary 모드를 선언하고 파일들을 전송(mput *)해야 합니다.
   2. ``mput *`` 실행 시 대화형 묻기 비활성화
      * 리눅스 환경에 따라 mput 실행 시 파일마다 y/n을 물어볼 수 있습니다. 만약 스크립트가 전송 중에 멈춘다면 user 명령어 다음 줄에 prompt 명령어를 한 줄 추가해 주시면 대화형 모드가 꺼집니다.
   3. 보안 주의
      * FTP는 SFTP와 달리 데이터와 비밀번호가 평문(Plain Text)으로 네트워크에 노출됩니다. 만약 외부 인터넷망을 통해 전송해야 한다면 보안을 위해 FTP보다는 가급적 SFTP나 FTPS(SSL/TLS 적용 FTP) 환경으로 전환하는 것을 강력히 권장합니다.
   4. prompt off (대화형 묻기 비활성화)
      * 리눅스 기본 ftp 명령어는 대량의 파일을 mput *으로 올릴 때, 파일 하나하나마다 mput file1.csv? (y/n)과 같이 사용자 확인을 요구합니다.
      * prompt off (또는 단독으로 prompt) 명령어를 입력하면 이 확인 절차가 꺼지며, 질문 없이 모든 파일이 즉시 자동으로 업로드됩니다.
   5. passive on vs passive off (전송 모드 설정)
      * FTP는 제어용 포트(21번) 외에 데이터를 주고받을 데이터 포트를 추가로 사용하는데, 이 포트를 누가 열어주느냐에 따라 모드가 갈립니다. [1]
      * Passive 모드 (passive on)
         - 방식: 서버가 데이터 포트를 열고, AP 서버(클라이언트)가 서버에 접속하여 데이터를 보냅니다.
         - 언제 쓰나요?: AP 서버가 AWS, NCP 같은 클라우드 환경에 있거나 엄격한 사내 방화벽 뒤에 있어서, 외부에서 AP 서버로 들어오는 접속(Inbound)이 차단되어 있을 때 사용합니다. 현재 대부분의 엔터프라이즈 환경 표준입니다. [2] 
      * Active 모드 (passive off)
         - 방식: AP 서버(클라이언트)가 데이터 포트를 열고, FTP 서버가 AP 서버로 역접속을 시도합니다.
         - 언제 쓰나요?: 타겟 FTP 서버가 외부 접속을 받지 못하도록 고정되어 있고, 반대로 AP 서버의 방화벽이 외부로부터의 접속을 전면 허용하고 있을 때 사용합니다. 만약 방화벽 때문에 Active 모드가 안 되면 425 Can't open data connection 같은 에러가 발생합니다.
